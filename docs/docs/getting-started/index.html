Getting started

Everything you need to run mc-pulse locally and explore the API.

Prerequisites​

  • Node.js 18+ (ESM project).
  • Postgres (for servers, snapshots, API keys, and webhooks).
  • Redis (Java status cache). The server will still start if Redis is down, but Java lookups will skip caching.

Configure environment​

Copy the example file and set your secrets:

cp .env.example .env

Key variables:

VariablePurposeDefault
PORTHTTP port for the API server3000
DATABASE_URLPostgres connection string (mc_pulse schema is used)required
REDIS_URLRedis connection stringrequired
CACHE_TTL_SECONDSRedis TTL for Java status cache30
DEFAULT_SERVER_HOSTFallback host when none is providedlocalhost
DEFAULT_SERVER_PORTFallback port for Java pings25565

Bedrock pings default to port 19132 if no port is provided.

Install and set up the database​

npm install
npx prisma generate
npx prisma migrate dev

migrate dev creates the mc_pulse schema and tables locally. Use npx prisma migrate deploy for production rollouts.

Run locally​

npm run dev

mc-pulse listens on http://localhost:${PORT} and exposes /health for quick checks.

Quick smoke test​

# Health
curl http://localhost:${PORT:-3000}/health

# Create an API key (save the returned key; there is no list endpoint)
API_KEY=$(curl -s -X POST http://localhost:${PORT:-3000}/api/api-keys \
  -H "Content-Type: application/json" -d '{"label":"local test"}' | jq -r .key)

# Java status lookup
curl "http://localhost:${PORT:-3000}/api/status/java?host=arch.mc"

# Notify all webhooks for this key (replace host/port as needed)
curl -X POST "http://localhost:${PORT:-3000}/api/notify?host=arch.mc&type=java" \
  -H "x-api-key: ${API_KEY}"

If you do not have jq, just copy the key value from the JSON response.

Next stop: read the API reference and webhook guide to wire this into your app or Discord bot.
